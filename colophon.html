how this site is built and why
philosophy

this site exists because i wanted a corner of the internet that's actually mine. not a profile on someone else's platform, not a link in a bio, not content that disappears if a company decides to shut down or change its algorithm. just a place i control, that works the way i want it to.

everything here is built on owned infrastructure — static files on github pages, no database, no backend, no monthly bill. if github disappeared tomorrow the whole thing is just a folder of html files i can host anywhere in five minutes.

i have a strong preference for things i own over things i rent. music on a dedicated mp3 player instead of spotify. files on disk instead of in the cloud. a website instead of a twitter bio. this site is that philosophy applied to having a presence online.

the stack is deliberately boring. no frameworks, no build steps, no node_modules. just html, css, and enough javascript to fetch some json. it loads fast, it works on any device, and i understand every line of it.

stack
hosting
github pages — free, deploys on push, zero config
frontend
vanilla html / css / js — no frameworks, no build step
data storage
json files in the repo — nows.json, blogposts/index.json, committed via github api
fonts
courier new for mono, system sans-serif for prose — no web font requests
music data
last.fm api — scrobbled via web scrobbler on desktop
server
rift anarchy runs on an ovh vps — paper mc, grim anticheat, nginx
tools
iNow
a pwa i built to post activity updates from my phone — commits directly to nows.json via github api built by me
blog editor
pin-gated in-browser editor at /blogposts/editor.html — markdown, tags, media embeds built by me
nifty
discord.js bot for rift anarchy — application management, season automation built by me
daily driver
cachyos linux with the cosmic desktop
editor
whatever's open — no strong opinions here
